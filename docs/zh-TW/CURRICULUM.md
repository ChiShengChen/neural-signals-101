# 課程綱要與規劃藍圖（v2 — 以學生為核心的重新設計）

[English](../CURRICULUM.md) · **繁體中文**

本文件是這份教學的規劃藍圖。**v1** 的目標對象是已熟悉 sklearn/PyTorch、只需要補充神經訊號知識的工程師。**v2** 重新鎖定真實的使用者群體：**初次接觸神經 AI/ML 的大學生**——他們在兩個面向上往往都不夠扎實：ML 基礎尚未內化、線性代數與訊號處理的直覺薄弱，就連 `(n_trials, n_channels, n_times)` 這樣的陣列形狀也可能是一道門檻。

## 為何重新設計（核心精神的延伸說明）

整份教學的主軸是**「不要欺騙自己」**。對工程師來說，自我欺騙的第一種形式是*資料洩漏*，所以 v1 可以直接從評估章節開始。但對初學者來說，洩漏甚至不是*第一個*陷阱。在它之前，有三個更早會碰到的問題：

1. **數字文盲** — 看著 `0.92` 卻不知道它相對於隨機猜測、`N=9` 位受試者、或信賴區間來說意味著什麼。
2. **形狀錯誤** — 搞錯了某個軸、靜默的廣播行為，然後*相信*了那些錯誤的輸出。
3. **容易被誇大宣傳牽著走** — 輕易相信「線下準確率 95%」或「EEG 讀心術」這類標題。

這些問題發生在洩漏*之前*，而 v1 默默地假設讀者已對此免疫。v2 將核心精神的建立提前，讓評估章節真正能落地生根。

## 課程支柱（優先等級）

- 🟥 **支柱（拓展受眾）：** ML 零基礎入門、視覺化數學直覺、神經生理學基礎、倫理與反誇大宣傳。*（使用者優先項目 1、2、4）*
- 🟧 **課程工程（教科書 → 課程）：** 每章的先修要求 + 難度標章、「預測後再執行」練習格、概念確認小測驗，以及附錄（詞彙表、疑難排解、練習題與解答）。*（使用者優先項目 3）*
- 🟨 **增強功能：** 統計直覺、遊戲化的隱藏測試集期末專題，以及即時處理／低成本硬體的動機引導。*（使用者優先項目 5、6）*

## v2.1 補充說明 — 主線 + 支線任務，以及誠實核心的深化

後期的一次精修（章節編號 00–15 維持不變，不重新排號）新增了以下內容：

- **`deep-dives/`** — **進階天花板**（CSP 幾何與推導、黎曼小樣本數學、統計嚴謹性、機率水準的信賴區間、基準測試過擬合、ICA/ASR 內部機制）。這實現了**「主線精簡慢行 + 支線任務深而硬」**的設計：5 分鐘 CPU 執行的承諾**只適用於主線**；deep-dives 內容可能更耗時，且**不包含在 CI smoke 測試中**。每個 deep-dive 章節都以一個更隱微的、主題專屬的陷阱作結，讓「不要欺騙自己」的精神延伸到最深的內容。
- **第 12 章重新定位** — 核心規則現在是*讓評估方式與你的應用宣稱相符*，而非「LOSO 是唯一的真理」：對一個**已校正的個人化模型**而言，沒有洩漏的 within-subject 評估是誠實的，只要清楚標示即可。（避免把隨機切分的迷思，換成另一種 LOSO 教條。）
- **陷阱 #7 — 在測試集上調整超參數或選擇模型**（巢狀交叉驗證）被新增為第七組 WRONG→RIGHT 對照：這是實務中最常見的靜默殺手。
- **第 09 章隨機種子變異誠實說明** — 單一隨機種子的執行結果只能教你*流程是否通暢*，不能用來比較模型架構；種子間的變異幅度會被明確展示。
- **第 11 章嚴謹化** — 加入二項式「高於機率水準」檢定，以及 Nadeau-Bengio 校正的重抽樣 t 檢定（未校正的 CV t 檢定過於保守偏差）。
- **CI 誠實守門員**（`tests/test_pitfalls.py`）— 用自動化測試確保 WRONG>RIGHT 的對照關係成立。
